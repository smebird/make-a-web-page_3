Lesson 1.1: Basics of the Web and HTML

What is the web?

The web is a bunch of computers that communicate with each other.

How does the web work?

The web has four main components:

  1. You: You want to visit a web page
  2. Your computer plus a Browser: Your browser makes a request via the internet
  3. The internet: Sends an HTTP request to a server
  4. Server: Locates HTML document and sends it to your computer where your web browser interprets and displays it on your screen

What is a web page?

A web page is a document written in a language called HTML. Web browsers read these documents, interpret and display them.

What is HTML?

HTML (HyperText Markup Language) is the heart of the web.

HTML is made up of:

  • Text content (What you see)
  • Markup (What the text looks like)
  • References to other documents (eg: images, videos, etc...)
  • Links to other pages
  • .
  • .
  • .

Basic HTML vocabulary you should know:

Element:

A structure in HTML that consists of an opening tag, content, and a closing tag.

Elements can be either inline (effecting just the text enclosed by tags), or block (block elements have an invisible box around them with height and width that separates them from any other element). While most Elements have both an opening and closing tag, some Elements use only one tag and are referred to as void.

Tags:

Tags mark the beginning and end of an Element. They are made of a less-than symbol, followed by the Element's name, followed by a greater-than symbol.

Attributes and values:

Attributes follow tags and point to a value. Examples are images and links to other web pages

A Couple Other Points to Remember:

Computers are stupid

Why? Because they interpret instruction literally. If you use the wrong Element name in your opening tag or make a typo, a computer cannot "know" what you intended. A program will only work if the instructions are clear and the grammar correct.

There Is Far Too Much to Remember

Don't try to memorize every single fact. Instead, concentrate on understanding how programming languages work, and rely on the internet to remind you of an Element's name or a particular structure.

What is CSS?

CSS = "Cascading Style Sheets"

Cascading refers to the process of moving from the most general property to the most specific property in your document.

The styles used in your project can be broken into separate CSS files or "sheets" to aid in organizing.

CSS properties, or rules, work in a hierarchy, much as laws function in a country:

  1. Federal = Document wide rules: properties applied to the highest level <div> block
  2. State = Properties applied to sub <div> blocks
  3. Municipal = Properties applied to sub-sub <div> blocks
  4. Home = Properties applied to individual elements

Cascading depends upon INHERITANCE:

INHERITANCE means that a property assigned to an element will be applied to all its descendents (children), unless
overwritten by a more local property.

Not all propterties are heritable. Refer to the CSS reference to find out whether a property is, or is not.

Understanding the Box Model

HTML boxes contain the following:

  1. Margin
  2. Border
  3. Padding
  4. Content

You can see an element's box in Dev Tools by clicking the Computed tab

At the center of this box is the Content.

  • The image or text that appears on the website
  • Consider the content the nucleus: It sits at the center and the surrounding layers protect it, so that other items don't overlap it or sit too close, making the website visually unappealing.

Next is the Padding.

  • The Padding clears the area around the Content.
  • It is affected by the background-color of the box
  • Think of it as the space inside the box that protects the Content directly.

Next is the Border.

  • The Border goes around both the Padding and the Content
  • It is inherited from the color property of the box.

Last, the out layer is the Margin.

  • The Margin clears an area around the Border
  • It has no background-color. It is completely transparent.
  • Think of it as the space between boxes

Calculating Box Size

Default = Sum of Border, Padding and Content

To make the calculation easier, we put the following code into our CSS file:

  1. * {
  2. box-sizing: border-box;
  3. }

Now, the calculation of the box includes both the border and the padding, so that it's easier to make layouts.

You just set the width of the overall box, and no matter how you tweak the padding and border layers, the size of the box will always stay the same.

The CSS must include browser-specific prefixes:

  • -webkit-box-sizing: border-box;
  • -moz-box-sizing: border-box;
  • -ms-box-sizing: border-box
  • box-sizing: border-box

You can set the size of your box layers in pixels or in percentages:

  • % changes the box as the screen size changes
  • Pixel sets the image to one size, regardless of device. This can lead to a bad user experience

You can set max-width, so that an element can get smaller, but not larger than a set value. Max-width won't change layout.

Lesson 2.1: Introduction to Serious Programming

What is Computer Science?

Computer Science is about how to solve problems by breaking them into smaller pieces and then mechanically and precisely describing the sequence of steps that you would use to solve each piece... and those steps can be executed by a computer.

What is a computer?

A universal machine that we can program to do any computation. Its power lies in doing things very fast: billions of instructions per second.

What is a program?

A very precise sequence of steps that tells the computer what to do. Anything we can write a program for, we can make a computer do.

To learn computing, we need to learn a programming language.

What is a programing language?

A vocabulary and set of grammatical rules for instructing a computer to perform specific tasks.

Programming languages come in three general forms:

Machine language:

  1. Written in numbers; the only language the computer really understands
  2. Platform specific. Machine language cannot be used on different kinds of computers.
  3. The fastest language to process

Assembly language:

  1. Uses the same instructions as in Machine Language, but keywords and variables have names rather than numbers only, making it easier to read and write.
  2. Like machine language, it is platform specific
  3. Assembly language must be translated into the machine language of a CPU by a program called an assembler.
  4. Very fast language to process

High Level Language:

  1. The closest to human language, it is easier to read, write and maintain.
  2. Platform independent: it can be used on any platform.
  3. Must be put through an interpreter or a compiler that translates the human-like language into the machine language of numbers.
  4. Slower language to process

See webopedia for more information.

Python is a High Level Program with advantages:

Python is easy to use:

It's the closest to human language, so easy to use that it's been called "programming at the speed of thought." Python programs are shorter and take less time to create than programs in many other popular languages.

Python is a "Glue" Language:

Python can be integrated with other languages such as C, C++ and Java. This means that a programmer can take advatage of work already done in another language while using Python. It also means that the strengths of other languages, such as the extra speed that C or C++ might offer.

Python has a strong community:

Python has the "Python Tutor mailing list, a more informal way for beginning programmers to ask questions The list is at: Python Tutor.

Python is also an interpreter:

It translates our code into machine language and that code can be run on any platform.

Why don't we write programs in human languages?

Natural (human) Languages are:

  • Ambiguous: Words and sentences are open to interpretation. (Example: what is the meaning of bi-weekly -- twice a week, or once every two weeks?).
  • Verbose: Natural languages use too many words and code made with them would be too bulky and slow to process.

How do programming languages work?

Like natural (human) languages, programming languages require Grammar:

Grammar:

A set of production rules that describe how to form strings from the language's alphabet that are valid according to the language's syntax.

  • String: A sequence of characters
  • Syntax: The combinations of symbols that are considered to be a correctly structured document or fragment in that language.

See the following Wikipedia pages for more information: Formal Grammar

How is human, or natural, language turned into programming language?

Using Replacement Grammars, or Context-Free Grammars, such as the Backus-Naur Form.

Replacement Grammars:

A notation used to describe each part of a language using agreed upon rules. A rule might look like the following:

  • <Sentence> ::= <subject> <verb> | <subject> <verb> <object>
  • .
  • (Which would be translated as: "A sentence consists of a subject and a verb OR a subject, verb and an object.")

In a sequence of steps, or derivations, each non-terminal (the words in < >) are gradually replaced by a terminal (such as <verb > replaced with the terminal "jump")

Replacement Grammars allow us to describe an infinitely large language with a small set of rules.

Languages made with Replacement Grammars are extremely precise!

Languages like Python, made with Replacement Grammars, are far more strict than natural languages. Rules must be followed exactly, or the instructions will not be understood or carried out.

For instance, when writing mathematical expressions in Python, you cannot mistype any part of the expression.

  • "2 + 2 _ 3" might be understood by another person as a typo for "2 + 2 + 3", but the Python interpreter will see it as an error.

Lesson 2.2: Variables and Strings

What is a Variable?

In computer programming the word variable denotes a storage location containing some value that is paired with an identifier (a symbolic name).

The value, or expression, that has been assigned to an identifier can be changed. This does not actually delete the former value from storage, but it is no longer paired to its original identifier.

If unpaired values are left in storage, programs can suffer "memory leaks" as storage reaches capacity, eventually leading to a program crash. There are Garbage Collection Modules available for Python that can be used to clear cluttered storage.

Python syntax and standard conventions:

In Python, the syntax for assigning a value to an identifier is as follows:

  • name = expression (where name is the identifier and expression is the value.)

Values can be either text characters, referred to as "strings," or numbers, referred to either as "integers" (without decimal points) or as "floating" (with decimal points)

In Python, any value/expression that is enclosed in quotation marks, whether single or double, will be interpreted as a string, as in the following examples:

  • myName = "smebird"
  • someNumbers = '0123456789'
  • text_plusNumbers = '0123456789 and "smebird"'

To use Python to do any calculations, numbers are entered as a value with no quotation marks, as in the following examples:

  • myAddition = 1 + 2 + 3000000000000
  • myMultiplication = 1234 * 4321
  • myExponents = 3**9

The convention for identifiers is to start with a lowercase character. You cannot begin an identifier with a number.

Python will print strings in a different color, making it is easy to catch errors, like a forgotton end quote.
